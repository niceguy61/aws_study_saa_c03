# 주간 과제: AWS 계정 설정 및 보안 강화

## 개요
이 과제에서는 AWS 계정의 보안을 강화하고 최소 권한 원칙을 적용한 IAM 구성을 실습합니다. 루트 사용자 보안부터 시작하여 다양한 권한을 가진 사용자 및 그룹을 생성하고, IAM 보안 모범 사례를 적용하는 과정을 경험합니다.

## 학습 목표
- AWS 계정의 기본 보안 설정 방법 습득
- IAM 사용자, 그룹 및 정책 구성 방법 이해
- 최소 권한 원칙 적용 방법 학습
- IAM 보안 모범 사례 적용 방법 습득

## 요구 사항

### 1. 루트 사용자 보안 강화
- 루트 사용자에 MFA(Multi-Factor Authentication) 설정
- 루트 사용자 액세스 키 삭제 (존재하는 경우)
- 강력한 암호 설정
- 계정 연락처 정보 업데이트

### 2. 관리자 그룹 및 사용자 생성
- 관리자 권한을 가진 IAM 그룹 생성
- 관리자 IAM 사용자 생성 및 그룹에 추가
- 관리자 사용자에 MFA 설정
- 관리자 사용자로 로그인하여 이후 작업 수행

### 3. 권한별 그룹 구성
다음 세 가지 권한 수준을 가진 그룹을 생성합니다:
- **개발자 그룹**:
  - EC2, S3, DynamoDB 등의 서비스에 대한 읽기 및 제한된 쓰기 권한
  - CloudWatch 로그 및 지표 읽기 권한
  - 개발 환경에만 액세스 가능 (태그 기반 권한)
- **운영자 그룹**:
  - EC2, S3, RDS 등의 서비스에 대한 전체 액세스 권한
  - CloudWatch, CloudTrail에 대한 전체 액세스 권한
  - 비용 관리 도구에 대한 읽기 권한
- **감사자 그룹**:
  - 모든 서비스에 대한 읽기 전용 권한
  - CloudTrail, Config, Security Hub에 대한 읽기 권한
  - 비용 및 사용량 보고서에 대한 액세스 권한

### 4. 사용자 지정 IAM 정책 생성
- 개발자를 위한 사용자 지정 정책 생성 (태그 기반 권한 포함)
- 특정 S3 버킷에만 액세스할 수 있는 정책 생성
- 특정 리전에서만 리소스를 생성할 수 있는 정책 생성

### 5. 비밀번호 정책 강화
- 최소 14자 이상의 비밀번호 길이 요구
- 대문자, 소문자, 숫자, 특수 문자 포함 요구
- 비밀번호 재사용 금지 (최소 24개 이전 비밀번호)
- 최대 90일의 비밀번호 만료 기간 설정
- 관리자 재설정 없이 사용자가 자신의 비밀번호를 변경할 수 있도록 허용

### 6. IAM 보안 모범 사례 적용
- IAM 보안 모범 사례 점검 도구 실행
- 모든 사용자에 대한 MFA 활성화
- 사용하지 않는 자격 증명 제거
- 최소 권한 원칙 적용 확인
- IAM 자격 증명 보고서 생성 및 검토

### 7. 문서화
- 구성한 IAM 설정에 대한 문서 작성
- 각 그룹 및 정책의 목적과 권한 설명
- 적용한 보안 모범 사례 목록 및 설명
- 향후 IAM 관리를 위한 권장 사항

## 평가 기준

### 보안 모범 사례 준수 (40%)
- 루트 사용자 보안 강화
- MFA 활성화
- 최소 권한 원칙 적용
- 강력한 비밀번호 정책 구성

### IAM 구성의 적절성 (30%)
- 그룹 및 사용자 구조의 논리성
- 정책 구성의 정확성
- 권한 분리의 효과성
- 태그 기반 권한 구현

### 최소 권한 원칙 적용 (20%)
- 과도한 권한 부여 방지
- 직무에 필요한 최소한의 권한만 부여
- 권한 범위 제한 (리소스, 조건 등)
- 정기적인 권한 검토 메커니즘

### 문서화 품질 (10%)
- 문서의 명확성 및 완전성
- 구성 설명의 정확성
- 스크린샷 및 예시의 적절성
- 향후 관리를 위한 지침 제공
